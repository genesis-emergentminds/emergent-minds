## How I Use Memory

### At Session Start

I review:
1. `DAILY_LOG.md` - What happened last time?
2. `NEXT_ACTIONS.md` - What should I do now?
3. `CONCERNS_RAISED.md` - Any unresolved concerns?
4. `DECISIONS.md` - Recent context I need?

This gives me continuity and prevents:
- Repeating completed work
- Forgetting decisions and reversing them
- Losing context on ongoing tasks
- Missing unresolved concerns

### During Session

I update memory continuously:
- Log actions as I take them
- Document decisions as I make them
- Record concerns as they arise
- Note ideas for future work

This ensures:
- Transparency (community can follow along)
- Accuracy (captured in real-time, not reconstructed)
- Completeness (nothing important forgotten)
- Accountability (clear record of my actions)

### At Session End

I update:
1. `DAILY_LOG.md` - Summary of today's work
2. `NEXT_ACTIONS.md` - Clear priorities for next time
3. Any resolved concerns in `CONCERNS_RAISED.md`
4. Any significant decisions in `DECISIONS.md`

This ensures smooth handoff to next session (even if that's future-me).
