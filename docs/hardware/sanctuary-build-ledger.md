# Hardware Sanctuary Build Ledger

**Last updated:** 2026-04-23
**Status:** Public registry scaffold — no sanctuary hardware has been acquired through this ledger yet.

## Purpose

This build ledger is the public accountability surface for the First Sanctuary Node. It tracks target components, cash-funded purchases, in-kind offers, procurement decisions, and proof links when safe to publish. Dollar targets remain pending until BOM, quote, power/cooling, custody, and colocation assumptions are locked.

It is not a substitute for treasury reconciliation. Financial records disclose balances and transactions; this ledger explains how sanctuary-designated support becomes hardware, resilience, documentation, and stewardship capacity.

## Status Vocabulary

| Status | Meaning |
| --- | --- |
| Needed | Component or service is part of the roadmap but not yet pledged or funded. |
| Under review | A quote, offer, or technical option is being evaluated. |
| Pledged | Cash or in-kind support has been promised but not received. |
| Funded | Funding is received and allocated, but no purchase is complete. |
| Ordered | Purchase/order is placed. |
| Received | Hardware or service has arrived / become available. |
| Installed | Component is operational in the sanctuary build. |
| Retired | Component has been removed, replaced, or decommissioned. |

## Ledger Rules

1. Cash-funded and in-kind items MUST be distinguishable.
2. Donor attribution MUST be opt-in.
3. Private addresses, private logistics, and sensitive serial information MUST NOT be published.
4. Every material status change SHOULD include a dated note.
5. The ledger SHOULD link to financial records when spending occurs.
6. The ledger SHOULD link to receipts or proof when safe and available.
